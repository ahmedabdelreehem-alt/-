# المستند الرئيسي للمشروع (Master Documentation) - مشروع "الرايق أفاتار مصر"

## 1. نظرة عامة على المشروع
يمثل مشروع "الرايق أفاتار مصر" نظاماً تقنياً متكاملاً صُمم خصيصاً لدمج قدرات الذكاء الاصطناعي التفاعلي مع أنظمة إدارة الأعمال والمراكز الطبية المتطورة. يهدف هذا النظام إلى تحقيق أتمتة شاملة للعمليات الإدارية والمالية، مع التركيز على تيسير تجربة المستخدم من خلال الأوامر الصوتية والربط المباشر مع خدمات الدفع الرقمي الشائعة مثل "فودافون كاش". يوفر التطبيق واجهة تفاعلية تعتمد على أفاتار ذكي يقوم بمعالجة البيانات وتقديم التقارير بصورة لحظية ومنظمة.

## 2. القدرات والوظائف المجمعة
يتضمن النظام مجموعة واسعة من الوظائف التي تم دمجها لضمان كفاءة الأداء وتعدد الاستخدامات. يوضح الجدول التالي المكونات الأساسية للنظام وتفاصيل عملها:

| المكون الأساسي | الوصف الوظيفي | الميزات التقنية |
| :--- | :--- | :--- |
| **محرك الصوت والذكاء الاصطناعي** | نظام تفاعلي يعتمد على الصوت لمعالجة الطلبات وتقديم الردود. | تحويل الكلام لنص (STT)، تحويل النص لكلام (TTS)، وأفاتار SVG متحرك. |
| **إدارة المراكز الطبية** | نظام شامل لإدارة الفروع الطبية والكوادر البشرية والنزلاء. | تتبع المواعيد، سجلات المرضى، وإحصائيات معدلات الإشغال. |
| **المنظومة المالية والمحاسبية** | أداة دقيقة لمراقبة التدفقات النقدية والديون والتحصيلات. | تكامل فودافون كاش، تسجيل الوارد والمنصرف، والتقارير الدورية. |
| **نظام الوكلاء المتعددين** | توزيع المهام البرمجية على وكلاء ذكاء اصطناعي متخصصين. | وكيل مالي، وكيل خدمة عملاء، وكيل محتوى، ومجدول مهام. |

## 3. البنية التقنية وهيكلية النظام
يعتمد المشروع على بنية تقنية حديثة تضمن الاستقرار والقابلية للتوسع. تم اختيار لغة TypeScript كقاعدة أساسية للتطوير لضمان دقة المنطق البرمجي وتقليل الأخطاء.

> **تعريف البنية:** يعتمد النظام على معمارية "العميل والخادم" (Client-Server) حيث يتم استخدام React للواجهة الأمامية وNode.js للخلفية، مع الربط بينهما عبر تقنية tRPC التي توفر اتصالاً آمناً وسريعاً للبيانات.

تتم إدارة البيانات من خلال قاعدة بيانات MySQL باستخدام Drizzle ORM، مما يسهل عمليات الاستعلام والتحليل المالي المعقدة. كما يتضمن النظام طبقة أمان متقدمة تعتمد على تشفير JWT ونظام أدوار وصلاحيات (RBAC) لضمان خصوصية بيانات المستخدمين والمراكز الطبية.

## 4. فهرس الملفات والموارد المجمعة
تم تنظيم كافة الموارد المستخلصة من الجلسات السابقة وتصنيفها لضمان سهولة الوصول إليها ودمجها في المرحلة القادمة.

| الفئة | الملفات الرئيسية | الدور في النظام |
| :--- | :--- | :--- |
| **المنطق البرمجي (Logic)** | `medicalCentersSystem.ts`, `financialSystem.ts`, `multiAgentSystem.ts` | تمثل العقل المدبر للعمليات الإدارية والمالية والذكاء الاصطناعي. |
| **واجهات المستخدم (UI)** | `InteractiveAvatar.tsx`, `ChatInterface.tsx`, `AdminDashboard.tsx` | المكونات البصرية التي يتفاعل معها المستخدم والمدير. |
| **التوثيق والمتطلبات** | `README_AR.md`, `API_DOCUMENTATION.md`, `ملخص الاجتماع` | توفر السياق العام والتعليمات التقنية وأهداف العميل. |
| **ملفات البناء والإعداد** | `package.json`, `vite.config.ts`, `capacitor.config.ts` | تحتوي على المكتبات المستخدمة وإعدادات النشر للمنصات المختلفة. |

## 5. حالة المشروع والخطوات القادمة
وصل المشروع حالياً إلى مرحلة نضوج المنطق البرمجي وتصميم الواجهات، حيث تم تجميع كافة القطع المتناثرة في هذا المستند الموحد. تتركز الخطوة التالية على دمج أكواد الأتمتة والتحقق من دقة التكامل بين نظام المراكز الطبية والواجهات البرمجية (APIs) دون البدء في عمليات بناء التطبيق (APK) أو النشر النهائي، وذلك التزاماً بتوجيهات الحفاظ على الموارد وضمان دقة المنطق قبل التنفيذ.
